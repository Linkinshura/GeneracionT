## Caso elegido: Profesor virtual para Estudiantes

### Razon de eleccion

Elegi este caso debido a su complejidad a la hora de recibir todas las solicitudes de los alumnos sin ser separadas

### Problema a Resolver 

Filtrar por temas las solicitudes de los alumnos para poder enviar los datos a un docente capacitado para ese tema, automatizando  esto generamos una organizacion mayor para los docentes, ademas se puede implementar el envio de un apunte de ese tema especificamente al alumno para que pueda repasar el tema primero tambien

### Diagrama

[Diagrama](https://github.com/Linkinshura/GeneracionT/blob/main/TrabajosPracticos/TrabajoPractico3/Diagrama.drawio.png)


### Herramientas Utilizadas

#### ChatGPT(IA)

Se encargara de manejar la Base de datos a la que se envia todas las solicitudes y filtrarlas por tema


#### Base de Datos

Base de datos donde se guardan las solicitudes y sus datos(urgencia y tema) para luego filtrar por tema y generar un informe con todos los alumnos que requieren ese tema para el envio de un apunte y asignar un turno para una reunion virtual con un docente

### Explicacion

#### Lectura del caso:

Leemos el caso y determinamos ciertas fallas o mejoras para su posterior cambio

#### Idear la automatizacion:

Se organiza una automatizacion posible para el caso(filtrado y envio de temas, asignacion de reuniones virtuales)

#### Diagrama:

Mediante un diagrama tratamos de organizarnos para saber el funcionamiento exacto del tema

#### Respuesta:

Se envia un informe con los datos y el profesor organiza el turno y su horario correspondiente, datos que luego envia a la IA y esta lo manda a todos los clientes que enviaron la solicitud de ese tema


### Beneficios:

La automatizacion de esto generaria una eficiencia mayor para organizar las solicitudes y ademas puede promover que ya antes de la reunion entiendan el tema o que puedan en la reunion misma aportar sus conocimientos y dudas ya resueltas o que otros no habian tenido en cuenta
